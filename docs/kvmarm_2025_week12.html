[PATCH v3 0/1] KVM: arm64: Map GPU device memory as cacheable

总体来看，本周的讨论集中在如何安全地实现可缓存映射，确保用户空间和内核之间的内存属性一致性，以及如何通过适当的检查机制来防止潜在的错误和安全隐患。

[PATCH RFC] KVM: arm64: PMU: Use multiple host PMUs

19 messages 3 participants Started: 2025-03-19
本邮件讨论的主题是关于在 KVM（Kernel-based Virtual Machine）中使用多个主机 PMU（性能监控单元）的补丁提案。补丁的目的是解决 Windows 系统在处理 PMU 时遇到的崩溃问题，尤其是在 PMCCNTR_EL0 停止计数的情况下。补丁建议结合多个主机 PMU，以确保 vCPU 能够在不同的物理 CPU 上运行而不导致计数错误。

在历史讨论中，参与者们讨论了当前实现的局限性，包括计数溢出处理不足和特性集的广告问题。补丁的初步讨论显示出对现有 PMU 实现的担忧，认为将多个 PMU 结合可能会导致不一致的行为。

本周的讨论中，参与者们对补丁的可行性和设计提出了不同看法。Akihiko Odaki 强调需要特殊处理固定的 CPU 周期计数器，以便 Windows 能够正常运行。Oliver Upton 提出了一种可能的解决方案，即在没有明确指定 vPMU 类型时，继续呈现默认 PMU，并确保固定的 CPU 周期计数器在任何 PMUv3 实现中都能正常工作。

然而，Marc Zyngier 对此表示反对，认为这种做法可能会引入新的 ABI（应用程序二进制接口）问题，并强调应该保持现有的行为不变。最终，讨论集中在如何在不破坏现有行为的前提下，改进 PMU 的实现，以满足实际使用需求。
