本邮件讨论的主题是关于 KVM（Kernel-based Virtual Machine）在 ARM64 架构上对递归虚拟化（Recursive NV）支持的补丁系列（PATCH v4 00/17）。该补丁主要集中在实现 VNCR（Virtual Nested Context Register）支持，特别是 VNCR_EL2 的虚拟化。

在历史讨论中，Marc Zyngier 提到，VNCR 的实现涉及到在 EL2 中为 L1（第一层虚拟机）准备 VNCR 页，并确保 L2（第二层虚拟机）能够访问该页。补丁系列的重点是管理 TLB（Translation Lookaside Buffer）以确保在需要时能够正确访问 VNCR 页。

本周的新讨论中，Marc Zyngier 提出了多个补丁，具体进展包括：
1. 添加了 VNCR_EL2 的布局和分配 VNCR 页的逻辑，以支持在 ARMv8.4-NV 系统上运行 NV 客户机。
2. 引入了一个新的地址转换辅助函数，以支持 VNCR_EL2 的虚拟化。
3. 处理 VNCR_EL2 触发的故障，并确保在 MMU 通知时能够正确处理 VNCR 页的无效化。
4. 实现了 TLBI（Translation Lookaside Buffer Invalidate）指令的处理，以确保在虚拟化环境中能够正确管理 TLB。

这些补丁的最终目标是使 KVM 在 ARM64 上的递归虚拟化功能更加完善，并提供给用户空间对 VNCR_EL2 的访问能力。整体而言，该系列补丁在功能上已接近完整，但仍需进一步优化和测试。
