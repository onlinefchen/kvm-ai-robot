[PATCH v4 01/28] KVM: arm64: Add a new function to donate memory with prot

35 messages 6 participants Started: 2025-09-09
本邮件讨论的主题是关于 KVM（内核虚拟机）在 arm64 架构下的内存捐赠功能的补丁（PATCH v4 01/28）。该补丁旨在添加一个新函数，以便在捐赠内存时能够指定权限（prot）。

在历史讨论中，参与者们主要关注如何确保捐赠的内存权限正确。Will Deacon 提出应当强制检查权限为 KVM_PGTABLE_PROT_RW，以避免不当的只读或只写捐赠。此外，Mostafa Saleh 讨论了捐赠 MMIO（内存映射 I/O）给 hypervisor 的必要性，并提出了对 pfn（页框号）进行多重检查的建议。

本周的新讨论中，Mostafa 对补丁进行了更新，决定将权限检查变得更加严格，以确保只有在 hypervisor 代码中才能进行此操作。参与者们还讨论了如何在 SMMU 驱动中处理缓存一致性问题，Jason Gunthorpe 提出应始终使用可缓存内存，以提高性能，并避免在快速路径中发出缓存无效操作（CMO）。总体而言，讨论集中在确保内存捐赠的安全性和性能优化上。

[PATCH v2 00/16] KVM: arm64: TTW reporting on SEA and 52bit PA in S1 PTW

27 messages 2 participants Started: 2025-09-15
本邮件讨论的主题是关于 KVM（Kernel-based Virtual Machine）在 ARM64 架构下的 S1 页表走查（PTW）和 52 位物理地址（PA）支持的补丁系列。

**原始补丁内容**：
Marc Zyngier 提出的补丁系列旨在解决在 S1PTW 故障时未能报告走查级别的问题，导致违反架构规范。补丁通过实现 S1 页表走查和支持 52 位 PA，增强了 KVM 的功能。

**之前讨论要点**：
在历史讨论中，Marc 提到当前代码仅支持 48 位地址，而未实现 52 位 PA，因此需要扩展 S1 PTW 以支持 LPA 和 LPA2。补丁的实现涉及多个方面，包括计算 52 位 TTBR 地址、调整最大输出地址、解耦输出地址与描述符等。

**本周新讨论和进展**：
本周的讨论中，Marc 提交了 16 个补丁，涵盖了多个方面的改进，如：
1. 计算 52 位 PA 支持状态。
2. 允许在非 NV 虚拟 CPU 上使用 S1 PTW。
3. 增加了过滤钩子以在每个走查级别进行处理。
4. 改进了故障注入机制，以便在 S1 PTW 故障时报告正确的级别。

Oliver Upton 对补丁进行了审查，提出了一些小的改进建议，并最终确认补丁已被应用到下一个版本中。这些补丁的实施将提升 KVM 对 ARM64 架构的支持，确保更符合架构规范。

**本周的新讨论、进展或结论**：
本周的讨论主要集中在补丁的细节和实现上。Eric Auger 对补丁中的默认值设定提出了疑问，并建议在下一个补丁中合并相关内容。Oliver Upton 则提交了与 KVM ARM64 的调试寄存器相关的修复补丁，并已被采纳。这些补丁解决了在嵌套虚拟化环境中调试寄存器处理不当的问题，确保了更好的兼容性和稳定性。
