[PATCH v10 00/43] arm64: Support for Arm CCA in KVM

44 messages 1 participants Started: 2025-08-20
本邮件线程讨论了针对 Arm Confidential Compute Architecture (CCA) 的 KVM 支持的补丁系列（PATCH v10 00/43）。以下是讨论的主要内容：

1. **原始补丁/问题内容**：
该补丁系列旨在为 KVM 添加对 Arm CCA 的支持，使得在 KVM 上运行受保护的虚拟机成为可能。补丁包括对相关 API 的实现、内存管理、以及与 RMM（Realm Management Monitor）的交互。

2. **之前讨论要点**：
之前的讨论集中在如何处理 Realm 的生命周期管理、内存分配、以及如何在 KVM 中实现对 RMM 的调用。参与者们对如何处理未委托的内存、如何管理大页映射、以及如何在 Realm 中处理 MMIO（内存映射输入输出）进行了深入探讨。

3. **本周的新讨论、进展或结论**：
本周的讨论主要集中在补丁的具体实现细节上，包括：
- 增强了对 RMM 的支持，允许 VMM 在 Realm 中设置和管理内存状态。
- 处理了 Realm VCPU 的加载和状态管理，确保在进入 Realm 时适当地处理寄存器和状态。
- 讨论了如何在 Realm 中处理 PSCI（Power State Coordination Interface）请求。
- 增加了对 SVE（Scalable Vector Extension）支持的检查和配置。
- 讨论了如何处理 Realm 的中断和计时器，确保在 Realm 中的中断不会导致立即返回到主机。
- 提出了对 KVM_CAP_READONLY_MEM 的限制，Realm 中不支持只读内存映射。

总的来说，本周的讨论推动了对 Arm CCA 在 KVM 中支持的实现进展，解决了多个技术细节问题，并为未来的补丁奠定了基础。

- 本系列补丁的目标是为 KVM 提供 ARM SMMUv3 支持，主要通过嵌套翻译和 MMIO 仿真来实现。补丁分为多个部分，涵盖了驱动的初始化、命令队列、流表的仿真等。

3. **本周的新讨论、进展或结论**：
- 本周的讨论主要围绕以下几个方面：
