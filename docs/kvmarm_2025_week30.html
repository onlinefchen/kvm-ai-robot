[PATCH v15 00/21] KVM: Enable host userspace mapping for guest_memfd-backed memory for non-CoCo VMs

70 messages 7 participants Started: 2025-07-17
本邮件讨论的主题是关于KVM（内核虚拟机）支持非CoCo虚拟机的guest_memfd内存映射的补丁系列（PATCH v15 00/21）。该补丁旨在允许主机用户空间映射由guest_memfd支持的内存，以简化虚拟机管理程序（VMM）如Firecracker的内存管理。

**历史讨论要点**：
1. Fuad Tabba 提出了多个补丁，主要包括去除对KVM_SW_PROTECTED_VM的依赖、重命名Kconfig选项以更准确地反映功能、引入新的函数以明确架构对guest_memfd的支持等。
2. 讨论中提到，guest_memfd的支持不仅限于私有内存，还扩展到共享内存的映射。

**本周新讨论进展**：
1. Xiaoyao Li 分享了在QEMU中实现gmem mmap支持的实验结果，发现虽然可以使用gmem作为普通内存，但在TDX虚拟机中存在限制，kvm_arch_supports_gmem_mmap()返回false。
2. Fuad Tabba 表示不希望支持TDX虚拟机的此功能，因为TDX使用了不同的内存管理范式。
3. 讨论中还提到，KVM不应强制检查用户空间传递的guest_memfd是否可映射，认为这是用户空间与虚拟机之间的问题。
4. Sean Christopherson 提出，KVM在处理共享内存时应确保一致性，但不应在创建内存槽时进行严格检查。

总体而言，讨论集中在如何有效地实现和管理guest_memfd的支持，以及在不同虚拟机类型（如TDX）中的具体应用和限制。

[PATCH v16 00/22] KVM: Enable host userspace mapping for guest_memfd-backed memory for non-CoCo VMs

60 messages 5 participants Started: 2025-07-23
本邮件线程讨论了一个关于 KVM（Kernel-based Virtual Machine）补丁系列的更新，主要集中在为非 CoCo 虚拟机启用基于 guest_memfd 的内存映射。

1. **原始补丁内容**：
该补丁系列的核心是为 KVM 提供对非 CoCo 虚拟机的 guest_memfd 支持，允许主机用户空间映射这些内存。这一功能旨在简化虚拟机管理程序（VMM）的设计，并增强安全性。

2. **历史讨论要点**：
在之前的讨论中，补丁的设计目标包括支持 Firecracker 等 VMM 完全基于 guest_memfd 运行虚拟机，移除主机内核对客体内存的直接映射，以减少潜在的安全攻击面。此外，补丁还涉及对 Kconfig 选项的重命名和清理，以更好地反映其功能。

3. **本周新讨论与进展**：
本周的讨论主要集中在补丁的具体实现和测试上，包括对 x86 和 arm64 架构的支持。参与者们讨论了如何处理 guest_memfd 的 mmap() 支持，确保在不同虚拟机类型下的兼容性。补丁中引入了新的能力标志 KVM_CAP_GUEST_MEMFD_MMAP，以便用户空间能够设置 mmap()，并更新了相关文档。此外，补丁还扩展了自测试用例，以验证 mmap 功能的正确性。整体上，讨论气氛积极，参与者对补丁的实现表示认可，并提出了一些改进建议。

总结而言，本周的讨论推动了 KVM 对 guest_memfd 支持的进一步完善，确保了其在不同虚拟机类型下的有效性和安全性。
