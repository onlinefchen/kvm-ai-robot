[PATCH 00/12] KVM: Make irqfd registration globally unique

14 messages 2 participants Started: 2025-04-01
本邮件讨论的主题是关于 KVM（内核虚拟机）中 IRQFD（中断请求文件描述符）注册的全局唯一性。Sean Christopherson 提出了一个补丁系列（共12个补丁），旨在确保一个事件文件描述符（eventfd）在整个系统中只能绑定到一个 IRQFD。

**原始补丁内容**：
补丁的核心是修改 KVM 的 IRQFD 注册机制，要求每个事件文件描述符只能绑定到一个 IRQFD。这是为了防止用户空间错误导致的潜在问题，尤其是在虚拟机迁移时。

**之前讨论要点**：
在历史讨论中，KVM 已经禁止将同一个事件文件描述符绑定到同一虚拟机的多个 IRQFD，但未对跨虚拟机的绑定进行限制。此补丁系列的目标是增强这一机制，以避免用户空间的错误配置。

**本周新讨论与进展**：
在本周的讨论中，Sean 提出了多个补丁，逐步实现了这一目标，包括：
1. 引入新的等待队列助手函数，以确保优先等待者的唯一性。
2. 禁止将多个 IRQFD 绑定到同一事件文件描述符。
3. 修改自测用例，确保事件文件描述符的创建和分配符合新的唯一性要求。

此外，讨论中还涉及了对现有代码的优化和清理，以提高代码的可读性和安全性。整体来看，本周的讨论集中在实现和验证补丁的细节上，以确保 KVM 的稳定性和安全性。

[PATCH v3 1/1] KVM: arm64: Allow cacheable stage 2 mapping using VMA flags

13 messages 6 participants Started: 2025-03-19
本邮件讨论的主题是关于 KVM（内核虚拟机）在 arm64 架构下允许使用 VMA（虚拟内存区域）标志创建可缓存的二级映射的补丁（PATCH v3 1/1）。该补丁旨在解决当前 QEMU 无法识别 VFIO（虚拟功能 I/O）创建的可缓存内存映射的问题。

在历史讨论中，参与者们探讨了如何处理不支持 FWB（Faulting Write Buffer）硬件的情况，建议在创建内存槽时进行检查，以避免在运行的虚拟机中出现问题。Jason Gunthorpe 和 Catalin Marinas 等人认为，应该在内存槽创建时拒绝不支持的请求，并讨论了如何在 KVM 和 VFIO 之间建立有效的通信。

在本周的新讨论中，Jason Gunthorpe 强调了在内存槽创建时进行保护性检查的重要性，以确保在没有 FWB 支持的情况下，立即失败请求，而不是在页面故障时再检查。他还指出，所有支持 CCA（Cache Coherency Architecture）的 ARM 系统可能都具备 FWB，因此继续沿用这一方向是合理的。此外，他提到必须阻止“回退到设备”的策略，以解决安全问题。

总体来看，讨论集中在如何确保 KVM 在处理可缓存内存时的安全性与兼容性，特别是在 VFIO 设备的上下文中。

[6.1 PATCH RESEND 00/12] KVM: arm64: Backport of SVE fixes to v6.1

13 messages 1 participants Started: 2025-04-04
本邮件线程讨论了针对 KVM（Kernel-based Virtual Machine）在 arm64 架构上 SVE（Scalable Vector Extension）相关问题的补丁系列，主要集中在将 SVE 修复回溯到 v6.1 版本。

1. **原始补丁内容**：补丁系列的目标是将 Mark Rutland 提出的 SVE/KVM 交互的修复回溯到 v6.1，涉及多个文件的修改，包括对 FPSIMD（Floating Point SIMD）状态的管理和 KVM 进入和退出时的状态保存。

2. **之前讨论要点**：历史讨论中提到，KVM 在处理 SVE 状态时存在一些问题，例如未能正确保存和恢复主机的 FPSIMD/SVE 状态，导致在使用 SVE 的虚拟机中出现崩溃等问题。之前的补丁尝试通过清除 TIF_SVE 标志来简化状态管理，但这在性能优化时可能会引发问题。

3. **本周新讨论与进展**：本周的讨论集中在多个补丁的提交上，包括：
- **清除 TIF_SVE 管理**：通过明确告知主机需要保存的寄存器状态，简化 KVM 的代码。
- **强制保存主机状态**：在加载虚拟 CPU 时，强制保存主机的 FPSIMD/SVE 状态，以避免使用过时的数据。
- **激活和计算 cptr_el2 陷阱**：在激活陷阱时重新计算 cptr_el2 的值，避免在每个 vCPU 结构中存储该值。
- **急切切换 ZCR_EL{1,2}**：在客人和主机之间切换时，确保 ZCR_EL{1,2} 的值正确，以避免潜在的 SIGKILL 信号。

这些补丁旨在提高 KVM 对 SVE 的支持，解决历史遗留问题，并优化性能。
