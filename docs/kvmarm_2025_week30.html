[PATCH v15 00/21] KVM: Enable host userspace mapping for guest_memfd-backed memory for non-CoCo VMs

70 messages 7 participants Started: 2025-07-17
本邮件线程讨论了一个关于 KVM 的补丁系列，主要目标是为非 CoCo 虚拟机启用主机用户空间映射的 guest_memfd 支持。补丁的主要内容包括对 KVM 配置选项的重命名和支持 guest_memfd 的新功能引入，以便简化内存管理并支持更多用例。

在历史讨论中，Fuad Tabba 提出了多个补丁，旨在消除对 KVM 的依赖关系并引入新的功能，例如 `kvm_arch_supports_gmem()` 和 `kvm_arch_supports_gmem_mmap()`，以明确指示架构是否支持 guest_memfd。这些补丁还包括对 KVM 配置选项的重命名，以更准确地反映其功能。

本周的新讨论集中在如何处理 QEMU 中的 guest_memfd 映射问题。Xiaoyao Li 提出在启用 gmem mmap 时遇到的挑战，特别是 TDX 虚拟机不支持此功能。Fuad Tabba 和其他参与者明确表示，不希望为 TDX 支持此特性，因为其架构与传统虚拟机不同。讨论中还提到，KVM 不会在创建内存槽时强制检查用户空间地址与 guest_memfd 的一致性，认为这是 VMM 的责任。

总的来说，讨论强调了 KVM 对 guest_memfd 支持的复杂性，以及如何在不同虚拟机类型之间保持一致性和功能性。

[PATCH v16 00/22] KVM: Enable host userspace mapping for guest_memfd-backed memory for non-CoCo VMs

60 messages 5 participants Started: 2025-07-23
本邮件线程讨论了一个关于 KVM（Kernel-based Virtual Machine）的补丁系列，主要集中在支持非 CoCo 虚拟机的 guest_memfd（访客内存文件描述符）内存映射功能。

1. **原始补丁内容**：
补丁系列的核心是启用主机用户空间对 guest_memfd 支持的内存进行映射。这一功能旨在简化虚拟机管理程序（VMM）的设计，并提高安全性，特别是在防止 Spectre 类攻击方面。

2. **历史讨论要点**：
之前的讨论集中在如何清理和重构 Kconfig 选项，以便更好地支持 guest_memfd。补丁中提到的主要变化包括将 CONFIG_KVM_PRIVATE_MEM 重命名为 CONFIG_KVM_GUEST_MEMFD，以更准确地反映其功能。此外，补丁还涉及到对 x86 和 arm64 架构的支持，确保它们能够正确处理 guest_memfd。

3. **本周的新讨论与进展**：
本周的讨论主要集中在补丁的实现细节和自测功能上。参与者们对补丁的各个部分进行了审查和反馈，确保其符合设计目标。特别是，补丁中引入的 mmap 功能得到了重点关注，确保其在不同虚拟机类型下的兼容性。此外，针对 guest_memfd 的自测功能也得到了扩展，以验证其在实际使用中的表现。

总的来说，这一系列补丁的目标是增强 KVM 对 guest_memfd 的支持，简化内存管理，同时提高安全性，并确保在不同架构和虚拟机类型下的有效性。
