[PATCH v9 0/6] KVM: arm64: Map GPU device memory as cacheable

31 messages 6 participants Started: 2025-06-21
本邮件线程讨论了一个关于 KVM（Kernel-based Virtual Machine）在 arm64 架构上将 GPU 设备内存映射为可缓存的补丁系列（PATCH v9 0/6）。该补丁旨在解决在 Grace Hopper 和 Blackwell 超级芯片平台上，CPU 可访问的缓存一致性 GPU 内存的映射问题。

在历史讨论中，Ankit Agrawal 提出了多个补丁，主要包括：
1. 将设备变量重命名为 `s2_force_noncacheable`，以更清晰地表明其功能。
2. 更新检测设备内存的检查逻辑。
3. 阻止可缓存的 PFNMAP 映射，以修复由于 S1 和 S2 映射属性不匹配而导致的安全漏洞。
4. 新增函数以确定硬件缓存管理支持。
5. 允许使用 VMA 标志进行可缓存的二级映射。
6. 暴露新的 KVM 能力，以指示是否支持可缓存的 PFNMAP。

本周的新讨论中，参与者们对补丁进行了审查并提出了改进建议。Ankit 提到他成功在虚拟机中分配了 GPU，并能够运行 `nvidia-smi` 命令，之前该命令在没有这些补丁的情况下会失败。Jason Gunthorpe 和 David Hildenbrand 对补丁的多个部分表示认可，并建议去掉一些不必要的检查，以简化代码逻辑。总体而言，补丁系列得到了积极的反馈，进一步的测试和审查正在进行中。
