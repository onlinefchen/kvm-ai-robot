[PATCH 0/3] KVM: arm64: Fix handling of ID_PFR1_EL1.GIC

8 messages 2 participants Started: 2025-10-13
本邮件线程讨论了针对 KVM（Kernel-based Virtual Machine）在 arm64 架构下的 ID_PFR1_EL1.GIC 处理的修复补丁。Marc Zyngier 提出了三个补丁，旨在解决 Peter Maydell 报告的问题：在恢复 GICv2 虚拟机时失败，原因是 ID_PFR1_EL1.GIC 不可写，而其 64 位等价物是可写的。

历史讨论中，之前的补丁（5cb57a1aff755）错误地将 ID_AA64PFR0_EL1.GIC 清零，导致在没有 GICv3 的情况下无法正确处理 ID 寄存器。Marc 的补丁系列包括：1）将 ID_PFR1_EL1.GIC 设为可写；2）在配置 GICv3 时直接设置 ID 寄存器字段；3）限制在用户空间 irqchip 中清除 ID 寄存器的操作。

本周的新讨论中，Marc 提到这些修复使得他能够在自己的 Synquacer 系统上成功保存和恢复 GICv2 虚拟机，表明补丁的有效性。此外，Ben Horgan 提出了与 set_id_regs 相关的清理补丁，进一步优化了测试代码，确保了测试的准确性和完整性。这些讨论和补丁的提出，显示了社区对 KVM arm64 支持的持续改进和关注。
